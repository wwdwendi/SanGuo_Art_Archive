# 三国服饰资料库 VI 与界面设计规范

本文件整理当前项目在多轮 UI 调整中形成的视觉规范。后续新增页面、修复样式或重构组件时，应优先按本文档检查，避免再次出现边框过多、颜色过亮、文字不可读、控件不统一、响应式错位等问题。

## 1. 设计目标

项目气质应是：

- 素雅、克制、沉稳。
- 以茶色、棕色、深墨色为主。
- 有历史资料库的秩序感，但避免厚重、笨拙、行政后台感。
- 深色页面需要有层次，但不能靠大量硬边框堆出来。
- 操作控件应精致、可读、统一，不能出现文字超框或状态颜色跳脱。

关键词：

- 深色茶棕
- 磨砂玻璃
- 低边框
- 低饱和高亮
- 轻量 panel
- 正文字体清晰
- 标题字体有文化感但不过粗

## 2. 当前用户反馈规律总结

### 2.1 去边框化，但不能完全无分区

多次反馈的核心不是“所有边框都删掉”，而是：

- 不要出现 card 外面还有 card、panel 外面还有 panel 的嵌套框体。
- 不要让每个区块都靠明显 1px 边框分割。
- 需要分区时，用低对比背景、透明度、blur、轻阴影来区分。
- 文献库曾被认为比较好，是因为 panel 有磨砂背景和层次，但没有粗重边框。

正确方式：

- 大容器：深色透明背景 + blur + 极弱内高光。
- 子区域：轻微背景差异或 radial highlight。
- 边框：只在必要的外层容器或交互控件上使用，透明度低。

错误方式：

- 多层卡片嵌套。
- 每个 section 都有明显边框。
- 亮白/亮灰色长条背景横穿页面。
- 高亮色太亮，破坏茶棕 VI。

### 2.2 深色模式要统一

用户明确要求资料库、图片库、时间线改成与首页、文献库一致的深色模式。

需要统一的页面：

- 首页
- 文献库
- 资料库
- 图片库
- 时间线

深色模式必须避免：

- 某个页面仍保留浅色 panel。
- 某个控件仍是白底。
- 同一组件在资料库和图片库样式不同。
- 铃铛、按钮、搜索框等跨页面颜色不一致。

### 2.3 资料库、图片库、时间线必须形成一组

这三个 tab 是同一视觉族群：

- 搜索 bar 样式必须一致。
- 筛选栏样式必须一致。
- 快捷筛选 chip 样式必须一致。
- 高亮色必须一致。
- panel 背景、间距、控件高度需要一致。
- 响应式逻辑需要一起验证。

修改其中一个页面的同类组件时，必须检查另外两个页面。

### 2.4 文献库是深色磨砂 panel 的参考

用户多次把文献库作为较满意的参考：

- panel 有背景 blur。
- 不靠重边框分割。
- 深色不死黑。
- 文字可读性较好。
- 控件状态相对克制。

后续资料库、图片库、时间线的 panel 应参照这个方向，而不是回到浅色卡片或硬边框。

### 2.5 字体要轻，不要特别粗

用户多次指出文字太粗。

当前字体策略：

- 正文使用正常正文字体。
- 大部分 UI 文案使用轻字重。
- 标题可以比正文重一个等级，但不能过粗。
- 标题字体使用 `SanJiZhuoSongTi-ZhongCu-2.ttf` 这一类更有文化感的宋体风格，但应控制字号和字重。
- 小标题、按钮、标签不要使用夸张粗体。

字重建议：

- 正文：300
- 常规强调：400
- 页面主标题：400 到 500 之间，视字体实际观感控制
- 按钮文字：300 或 400
- 不推荐：600、700 以上的大面积使用

### 2.6 高亮状态必须清透、低饱和、可读

多次反馈指出：

- 高亮色太亮。
- 高亮背景与文字撞色，看不清。
- 图片上叠加文字时，白色图片会影响阅读。
- 选中文字状态辨识度不足。

正确方式：

- 高亮使用低饱和茶棕、赤褐、暖灰。
- active 状态可以更深，但不能亮黄、亮橙、亮金。
- hover 只轻微提高亮度或透明度。
- 文本和背景需要有足够对比度。
- 在图片或复杂背景上，文字底部必须有暗色磨砂或渐变遮罩。

Bad example 必须避免：

- 深色列表里使用 `rgba(255,255,255,0.45)` 以上的大面积浅灰 hover 块。
- hover 后背景整体变亮，但标题、正文、元信息仍是低透明度灰字，导致文字发灰、发糊。
- 仅靠“变亮背景”表达选中或 hover，而没有同步保证标题、正文、标签的对比度。
- 奶白色 pill 或 badge 直接放在深色列表里，会比正文更抢眼，破坏信息层级。

正确替代：

- 深色列表 hover 使用低饱和茶棕透明扫光，例如 `rgba(151, 109, 69, 0.08 - 0.14)`。
- hover 背景应保持暗底，不使用整块浅灰或白灰色。
- 标题文字至少保持 `rgba(246, 236, 218, 0.86)` 以上；正文不低于 `rgba(235, 224, 207, 0.68)`。
- 元信息可以弱化，但不要低到无法阅读；建议 `rgba(235, 224, 207, 0.54 - 0.62)`。
- 标签、badge 使用深色磨砂或茶棕半透明底，不使用高亮奶白底。

### 2.7 按钮必须统一

用户反复要求检查全部 button：

- 大小统一。
- 高亮状态统一。
- hover 状态统一。
- 点击状态统一。
- 阴影状态统一。
- 不要文字超框。
- margin 不要过窄。
- 不要奇怪配色。

按钮设计原则：

- 茶棕深色为主。
- 低边框或无边框。
- 圆角适中。
- 高度偏紧凑，普通操作按钮约 36px，主要 CTA 约 40-42px，工具/标签型按钮约 28-32px，筛选分组标题约 52px。
- 文本不换行时必须保证宽度足够；必要时使用自适应宽度。
- 工具按钮优先 icon + tooltip 或 icon + 简短文字。
- 禁用状态降低透明度，不要完全消失。

### 2.8 搜索框必须统一且去边框化

资料库和图片库搜索 bar 被多次要求保持一致。

当前要求：

- 资料库搜索 bar 和图片库搜索 bar 必须时刻保持一致。
- 高度、宽度、背景、blur、阴影、圆角、输入框高度、placeholder 样式一致。
- 移动端不能横向溢出。
- 搜索框应是深色磨砂 pill，不应是白底硬边框。
- 浮动搜索框必须在页面内容中心点，不是浏览器含滚动条的中心点。

### 2.9 筛选栏要精致，不能 bulky

用户明确指出左侧筛选 panel 曾被改得笨重。

筛选栏规范：

- panel 背景使用深色磨砂。
- 内部分组不要过多硬边框。
- 展开/收起工具行靠左，图标文字垂直居中。
- checkbox、chip、下拉列表使用统一茶棕状态。
- 筛选列表和右侧 container 高度需要视觉对齐。
- 移动端筛选不能挤压主体，也不能横向溢出。

### 2.10 下拉列表不能被 panel 遮挡

所有下拉列表需要检查：

- z-index 足够高。
- 菜单背景符合深色 VI。
- 不被父 panel overflow 裁切。
- hover 和 selected 状态清晰可读。

### 2.11 响应式是重点风险

用户多次指出手机版本问题很大。

响应式必须检查：

- 侧栏与内容区在移动端重新堆叠。
- 搜索 bar 不偏移、不溢出。
- 按钮文字不超框。
- 筛选 panel 不挤压主体。
- 高度不要超出视口导致主要内容不可见。
- 横向滚动必须尽量消除。

每次改资料库、图片库、时间线布局后，都要检查：

- 1280px 桌面
- 390px 手机
- 720px 左右断点

### 2.12 状态栏、工具栏不要出现整条浅色带

用户指出时间线类别切换区域出现“白色条”。

规范：

- chip/button 组可以浮在背景上。
- 容器本身不要有全宽浅色条。
- 如果需要容器感，只能用极弱背景，不应横穿页面形成视觉噪音。

## 3. 色彩规范

### 3.1 主色

深色背景：

- 近黑茶色：`rgba(18, 16, 14, 0.78)`
- 深棕黑：`rgba(20, 18, 16, 0.58 - 0.72)`
- panel 深底：`rgba(25, 21, 18, 0.72)`

茶棕强调：

- 铜棕：`#9a7a55`
- 深茶：`#5f4430`
- 暖棕：`#8f6842`
- 低亮金棕：`rgba(224, 193, 150, 0.12 - 0.34)`

正文文字：

- 主文字：`#f4ead8` / `#eadfce`
- 次级文字：`rgba(235, 224, 207, 0.72 - 0.82)`
- 弱提示：`rgba(235, 224, 207, 0.48 - 0.58)`

### 3.2 禁止或少用

避免：

- 大面积纯黑。
- 大面积纯白。
- 亮黄、亮橙、亮金。
- 高饱和紫、蓝紫渐变。
- 明显亮灰横条。
- 白色卡片残留在深色页面里。

### 3.3 高亮色

active 推荐：

- `rgba(116, 82, 50, 0.5)`
- `rgba(160, 115, 72, 0.22)`
- `linear-gradient(180deg, rgba(82, 59, 40, 0.9), rgba(54, 39, 29, 0.94))`

hover 推荐：

- `rgba(151, 109, 69, 0.12 - 0.16)`
- `rgba(224, 193, 150, 0.08 - 0.13)`

选中文字：

- 深底页面：茶棕半透明背景，文字用深墨或浅米色，保证对比。

## 4. 材质与层级

### 4.1 磨砂玻璃

推荐 panel 样式：

```css
background:
  linear-gradient(180deg, rgba(255, 248, 236, 0.04), rgba(255, 248, 236, 0.014)),
  rgba(18, 16, 14, 0.66);
box-shadow:
  inset 0 1px 0 rgba(255, 248, 236, 0.035),
  0 14px 34px rgba(0, 0, 0, 0.16);
backdrop-filter: blur(18px) saturate(1.04);
```

### 4.2 边框

边框透明度建议：

- 外层 panel：`rgba(224, 193, 150, 0.07 - 0.12)`
- 控件 hover：`rgba(224, 193, 150, 0.2 - 0.3)`
- active：`rgba(214, 186, 143, 0.24 - 0.34)`

避免：

- `rgba(..., 0.4+)` 的大面积边框。
- 多层 panel 都有边框。
- dashed 边框用于主要空状态。

### 4.3 阴影

阴影要轻：

- 常规 panel：`0 10px 26px rgba(0, 0, 0, 0.14)`
- 浮层/dropdown：`0 18px 34px rgba(0, 0, 0, 0.26)`
- 不要强烈外发光。

## 5. 字体规范

### 5.1 字体角色

标题：

- 使用项目标题字体，例如三极宋体类字体。
- 用于页面主标题、重要板块标题。
- 不用于所有小控件。

正文：

- 使用 `Noto Sans SC` / `Source Han Sans SC` / `PingFang SC` 体系。
- 信息密集页面优先清晰可读。

英文或编号：

- 可使用更稳的 serif 或系统字体，但要保证与中文协调。

### 5.2 字重

- 正文：300
- 普通按钮：300 或 400
- 卡片标题：400
- 页面标题：400 到 500
- 禁止大量粗体。

## 6. 组件规范

### 6.1 按钮

按钮高度建议：

- 小按钮：28-32px
- 常规按钮：34-36px
- 重要操作：40-42px
- 筛选分组标题：约 52px

按钮状态：

- normal：透明或低透明茶棕背景。
- hover：轻微加深。
- active：深茶棕背景。
- disabled：降低文字透明度，背景极弱。

必须避免：

- 文字超框。
- 高亮色过亮。
- hover 后文字不可读。
- 同页面按钮高度不一致。

### 6.2 搜索框

浮动搜索框：

- pill 圆角。
- 深色磨砂。
- 资料库和图片库必须共用规则。
- 高度应轻巧，不占用过多首屏。
- 位置基于页面内容中心，不基于含滚动条的浏览器中心。

普通搜索框：

- 去边框化。
- 背景用深色透明。
- focus 用低饱和茶棕光圈或轻微边框。

### 6.3 筛选 panel

结构：

- 顶部标题区。
- 展开/收起工具行。
- 分组项。
- 选项列表。

要求：

- 工具行靠左。
- 工具行图标、文字、分隔符垂直居中。
- 分组标题不要过粗。
- checkbox 状态要清晰。
- 选中项不要亮色。

### 6.4 Chip / Tab

Chip：

- 独立浮动，不要有全宽浅色条。
- active 是深茶棕，不是亮色。
- hover 轻微增亮。

Tab：

- 同组 tab 高度一致。
- 文案不能挤压。
- 不要用大面积高亮条。

### 6.5 Dropdown

Dropdown 菜单：

- 深色磨砂背景。
- z-index 高于 panel。
- 不被 overflow 裁切。
- selected 状态低饱和。
- hover 状态清晰但不刺眼。

### 6.6 卡片

卡片规范：

- 圆角可以有，但不要过大。
- 结果卡片不要厚重。
- 图片卡片文字叠在图片上时必须有暗色遮罩。
- card 之间靠间距和背景层级区分，不靠重边框。

## 7. 页面规范

### 7.1 首页

首页是深色品牌气质参考：

- 茶棕深色。
- 标题文化感。
- 视觉层级更强。
- 不要让后续页面偏离首页基调。

### 7.2 文献库

文献库是磨砂 panel 的主要参考：

- panel 背景层次好。
- 深色阅读性较好。
- 不能被回退成边框化或浅色卡片。

### 7.3 资料库

重点：

- 深色模式完整。
- 搜索 bar 与图片库一致。
- 筛选栏不能 bulky。
- 结果卡片右侧按钮不能文字超框。
- 更多菜单应使用深色磨砂，不要白底。

### 7.4 图片库

重点：

- 搜索框与资料库一致。
- 筛选栏与资料库一致。
- title 与中心内容距离要协调。
- 图片高亮背景更清透。
- 图片上的文字必须可读。

### 7.5 时间线

重点：

- 深色模式完整。
- 类别 chip 不要有全宽浅色条。
- 时间线需要横线辅助结构。
- timeline card 需要去边框化但保持层级。
- 筛选栏、下拉、按钮要符合茶棕 VI。

## 8. 响应式规范

### 8.1 桌面

检查：

- panel 与右侧 container 高度视觉对齐。
- 搜索 bar 居中。
- 下拉不被遮挡。
- 按钮文字不超框。

### 8.2 移动端

检查：

- 不允许横向溢出。
- 搜索 bar 不偏移。
- 筛选 panel 不占据过多高度。
- 按钮可以换行或缩短，但不能超框。
- 页面主要内容必须能正常阅读。

### 8.3 断点

至少检查：

- 1280px 桌面
- 720px 断点
- 390px 手机

## 9. 可读性与可访问性

必须检查：

- normal、hover、active、disabled 下文字都可读。
- 高亮背景不压文字。
- 图片上的文字有遮罩。
- selected 文本对比度足够。
- accessible name 不应重复到影响自动化定位。
- 按钮实际功能与文本一致。

## 10. 修改验收清单

每次 UI 修改后，至少检查：

- 是否引入新的硬边框。
- 是否出现 panel 套 panel。
- 是否出现白底或浅色条残留。
- 是否符合茶棕深色 VI。
- hover/active/disabled 是否可读。
- 按钮文字是否超框。
- 资料库、图片库、时间线的同类组件是否一致。
- 移动端是否横向溢出。
- 下拉菜单是否被遮挡。
- 构建是否通过。
- 必要时用浏览器测量实际 computed style，而不是只看 CSS。

## 11. 推荐实现策略

优先：

- 使用共享 CSS 变量和共享 selector。
- 对资料库、图片库、时间线建立同组覆盖规则。
- 在 CSS 末尾保留明确注释，说明哪些组件必须保持一致。
- 小范围覆盖，不做无关重构。

避免：

- 为单个页面临时写孤立样式。
- 修改一个页面后不检查同组页面。
- 通过增加更多边框解决层级问题。
- 用亮色解决高亮问题。
- 只在桌面端验证。

## 12. 当前 VI 一句话定义

以深茶棕为主色、低饱和铜棕为强调色、磨砂玻璃为主要层级语言，通过轻边框、低阴影、清晰正文字体和克制高亮，建立一个素雅、可读、精致的三国服饰资料管理界面。
